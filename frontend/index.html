🧠 AI Model Architecture
1
Data Ingestion & Correlation
Cybersecurity telemetry (SIEM events, endpoint alerts, network logs) paired with transaction records by user_id and time window.
2
Feature Engineering
6 key features extracted: transaction amount, failed logins, geo-mismatch, cross-border flag, bytes transferred, velocity score. Normalized via StandardScaler.
3
Isolation Forest Anomaly Detection
200 trees, 8% contamination rate. Converts decision function output to 0–100 risk score. No labels needed — unsupervised, works on unknown threats.
4
Behavioral Rule Overlay
4 expert-defined rules add contextual boosts to scores. Rules catch patterns the statistical model might miss (e.g., privilege escalation before large transfer).
5
Explainable AI Output
Feature contributions (SHAP-style) + plain-language explanation + recommended action generated per alert. Analysts understand exactly WHY an alert was raised.

Why Explainability Matters in Banking:
  • Regulators (RBI, SEBI, Basel) require audit trails for AI decisions
  • Security analysts need to understand AI alerts to act on them
  • Black-box models erode trust — explainability builds it
  • Reduces false-positive fatigue by showing WHY an alert is real
False Positive Reduction Strategy:
  • Isolation Forest: unsupervised, no label bias
  • Rule overlay: context-aware boosting (not blanket alerts)
  • Multi-signal correlation: single signals ignored, combinations flagged
  • Threshold tuning: contamination=8% calibrated to reduce noise
